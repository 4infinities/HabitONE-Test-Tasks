# Аналіз когорт, Retention та Email Performance — Методологія

## Методологія

### 1. Загальний підхід

Рішення базувалося на аналізі 44 історичних **когорт** користувачів із побудовою прогнозів на горизонт у 52 тижні. Основною задачею було спрогнозувати **retention**, динаміку e-mail-розсилок та пов'язані метрики ефективності для когорт, дані по яких ще були неповними.

У процесі аналізу використовувався підхід когортного моделювання, де кожна когорта розглядалася окремо, а потім результати агрегувалися для отримання загальних прогнозних показників. Основна увага приділялася стабільності week-over-week retention та поведінці **CTR** у часі.

---

### 2. Аналіз retention та прогнозування когорт

На першому етапі було проаналізовано динаміку retention між тижнями (week-over-week retention). Аналіз показав, що після першого тижня значення retention стають відносно стабільними. Це дозволило застосувати просту kогарифмічну лінійну регресію для прогнозування подальшої поведінки когорт.

Для кожної когорти було розраховано week-over-week retention, після чого визначено середній коефіцієнт утримання користувачів. Саме цей коефіцієнт використовувався для апроксимації retention на наступні тижні.

Під час підготовки даних було виконано очищення аномалій:

- останній доступний тиждень кожної когорти було виключено з аналізу через систематично занижені значення;
- для когорти W15 було виключено початковий період, де retention перевищував 100%, оскільки ця когорта мала аномально високу кількість інсталяцій та нетипову поведінку.

Додатковий аналіз показав, що для перших приблизно 30 когорт week-over-week retention є стабільним, однак для молодших когорт цей показник починає поступово знижуватись. Водночас у старших когорт стабілізація retention відбувається приблизно з четвертого тижня.

На основі цього було сформульовано гіпотезу про існування стабільного ефекту «старіння» когорт. Для врахування цього ефекту було виконано лінійну апроксимацію самого коефіцієнта week-over-week retention. Це дозволило скоригувати прогноз для молодших когорт та оцінити їхню майбутню поведінку.

> Фінальна модель дозволила побудувати прогнозну таблицю retention на 52 тижні. Надійність моделі підтверджувалась низькою варіативністю коефіцієнтів: співвідношення стандартного відхилення до середнього значення не перевищувало 5%.

---

### 3. Моделювання e-mail розсилок

Окремо було проведено аналіз e-mail-розсилок по когортах. Було встановлено, що кількість e-mail прямо пропорційна кількості активних користувачів у когорті. Користувачі, які переставали бути активними, більше не враховувались у розсилках.

Також було визначено, що weekly e-mail rate на одного користувача є однаковим усередині всіх когорт і змінюється залежно від етапу **lifetime value** користувача:

- W1-W10 — 50 e-mails;
- W11-W23 — 70 e-mails;
- W24-W31 — 150 e-mails;
- W32+ - 100 e-mails.

> Таким чином, прогноз кількості майбутніх e-mail будувався на основі: прогнозного retention, прогнозної кількості активних користувачів та стабільного weekly e-mail rate.

---

### 4. Аналіз та прогнозування CTR

Паралельно було проаналізовано **CTR** (click-through rate) для всіх когорт. Аналіз показав, що CTR демонструє майже лінійне зростання з кожним наступним тижнем.

Попри певні відмінності між когортами, темпи цього зростання були достатньо схожими, що дозволило виділити кілька типових груп когорт із подібною поведінкою. Для кожної групи було побудовано лінійну апроксимацію CTR.


| Cohorts | Initial CTR, % | CTR growth rate WoW, % |
| ------- | -------------- | ---------------------- |
| W1-W10  | 3              | 0.3                    |
| W11-W24 | 2.1            | 0.21                   |
| W25     | 1.5            | 0.15                   |
| W26-W37 | 0.8            | 0.08                   |
| W38-W44 | 2.2            | 0.22                   |


> Додатково було помічено, що стартовий CTR молодших когорт поступово знижується порівняно зі старшими когортами. Через це прогнозний середній CTR для молодших когорт на річному горизонті також виявився нижчим. Цей ефект був врахований у фінальній моделі прогнозування.

---

### 5. Загальна логіка фінальних розрахунків

Усі фінальні оцінки базувалися на поєднанні:

- прогнозного retention;
- прогнозної кількості активних користувачів;
- моделі e-mail-розсилок;
- прогнозного CTR.

Такий підхід дозволив отримати консистентну систему прогнозів, яка одночасно враховує:

- природний **churn** користувачів;
- старіння когорт;
- зміну інтенсивності e-mail-комунікації;
- зміну ефективності взаємодії користувачів із розсилками.

## Завдання та відповіді

### Завдання 1. Скільки листів у середньому надіслано одному користувачу:

а) за 3 місяці з моменту встановлення додатку?
б) за 6 місяців з моменту встановлення додатку?
в) за 1 рік з моменту встановлення додатку?

Для розрахунку середньої кількості e-mail було використано два підходи.

**Кумулятивний blended-підхід** враховував усіх користувачів когорти, включно з тими, які відпали на ранніх етапах. Сумарна кількість e-mail розподілялась на всю початкову аудиторію. У результаті середня кількість e-mail на одного користувача поступово знижувалась зі збільшенням горизонту аналізу, оскільки менша кількість користувачів доживала до пізніх тижнів.

**Розрахунок лише для активних користувачів** враховував лише тих користувачів, які залишалися активними до певного моменту, наприклад до третього місяця. У такому випадку середня кількість e-mail на одного користувача зростала, оскільки активні користувачі продовжували отримувати розсилки на пізніх етапах lifecycle.

Фінально було розраховано обидва показники та наведено їх у підсумковій таблиці.

**Відповідь:**


|           | Emails per ANY user | Emails per ACTIVE user |
| --------- | ------------------- | ---------------------- |
| 3 months  | 93.80               | 1117.53                |
| 6 months  | 92.72               | 2302.17                |
| 12 months | 91.35               | 4394.40                |

### Завдання 2. Скільки в середньому «живе» один користувач додатку?

**Lifetime value** одного користувача було оцінено як сукупне середньозважене значення на основі всіх когорт.

Для цього використовувались:

- retention користувачів;
- початкова кількість встановлень для кожної когорти.

**Відповідь:** 14.59 weeks

---

### Завдання 3. Виходячи з ціни за відправку листа в $0.1, скільки коштує утримання користувача за допомогою email розсилки впродовж одного року?

Для розрахунку було використано кумулятивний blended-показник із першого завдання — середню кількість e-mail, яку один користувач отримує протягом року.

Такий підхід дозволяє автоматично врахувати:

- користувачів, які відпадають на ранніх етапах;
- користувачів, які залишаються активними до кінця року;
- зміну обсягів активної аудиторії в часі.

> Середня кількість e-mail множилась на вартість одного e-mail та прогнозний retention. Таким чином, фінальна оцінка вже містила в собі повний ефект churn користувачів та зміни активної аудиторії протягом року.

**Відповідь:** $9.14

---

### Завдання 4. Який середній CTR впродовж року для користувачів, які зареєструвались у проміжок:

а) з 17 по 20 тиждень?
б) з 21 по 24 тиждень?
в) з 25 по 28 тиждень?

Середній CTR було розраховано з використанням лінійних апроксимацій, побудованих на історичних даних когорт.

У моделі враховувались:

- щотижневе лінійне зростання CTR;
- поділ когорт на кластери зі схожою поведінкою;
- поступове зниження стартового CTR у молодших когорт.

> Через зниження стартового CTR у новіших когортах прогнозний середній CTR на річному горизонті також ставав нижчим. Цей ефект був інтегрований у фінальний прогноз.

**Відповідь:**


|            | Annual average CTR |
| ---------- | ------------------ |
| Week 17–20 | 5.47%              |
| Week 21–24 | 5.44%              |
| Week 25–28 | 2.65%              |
